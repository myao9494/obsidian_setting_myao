# obsidian_setting_myao

obsidianのセッテイングのリポジトリです。

## ベース

[GitHub - amatzk/obsidian-dagnetz-system: A simple Obsidian vault template that preserves thought evolution through editing restrictions and creates a DAG-structured second brain.](https://github.com/amatzk/obsidian-dagnetz-system)

これをベースとしている。

## 追加ルール

以下のタグだけは、編集 を許容する

- reference

これには、git command などのチートシート、アプリ開発の仕様書 などを想定している。とにかく、思考プロセスの履歴を必要としないもの。もしかしたら、本来は別のツールで管理した方がよいのかも。。。

## 理由

守破離でいきたいところであるが、これを辞書的なものでやると、大変煩雑になる。
また、辞書は、思考のプロセス履歴を取る必要がない。
なぜなら、
- 辞書のプロセスから、アイデアは生まれない

あくまで、このシステムは、思考プロセスを残すこと、ネットワークとして思考をつなげるものとして機能すると考える。
そういう意味合いでは、「文献」と近いものと考えられる。
むしろ、自分の運用では、「文献」の方が不要かもしれない
